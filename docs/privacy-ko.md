# GalaxyPods 개인정보처리방침

**시행일.** 2026년 5월 15일
**최종 개정일.** 2026년 5월 15일

GalaxyPods(이하 "본 앱")는 개인정보 보호법, 위치정보의 보호 및 이용 등에 관한 법률 등 관련 법령을 준수하며, 사용자의 개인정보를 소중히 다룹니다. 본 방침은 본 앱이 어떤 정보를 어떤 목적으로 처리하는지, 사용자가 어떤 권리를 행사할 수 있는지를 알리기 위해 작성되었습니다.

---

## 1. 개요 — 한 문장 요약

**본 앱은 사용자의 어떠한 개인정보도 외부 서버로 전송하지 않으며, 모든 데이터는 사용자의 단말 내에서만 처리·보관됩니다.**

---

## 2. 처리하는 정보의 항목 및 방법

### 2.1 본 앱이 처리하는 정보

| 항목 | 처리 목적 | 처리 위치 | 보관 기간 |
|---|---|---|---|
| Bluetooth Low Energy 광고 패킷 (제3자 기기인 무선 이어폰의 모델·배터리·착용 상태) | 화면 표시, 자동 재생/정지 | 사용자 단말 메모리 | 화면 표시 후 즉시 삭제 (영속화 X) |
| 마지막으로 이어폰과 연결이 끊어진 시점의 위치 좌표 (선택) | 분실 방지 — "마지막 위치 보기" 기능 제공 | 사용자 단말 내 DataStore | 사용자가 수동 삭제하거나 앱 삭제 시까지 |
| 앱 설정값 (귀감지 ON/OFF, 테마, 언어) | 사용자 설정 유지 | 사용자 단말 내 DataStore | 앱 삭제 시까지 |

### 2.2 본 앱이 처리하지 **않는** 정보

- 이름, 전화번호, 이메일, 생년월일 등 신원 식별 정보
- 결제 정보, 카드 번호
- 주소록, 통화 기록, 메시지
- 이어폰을 통해 재생되는 음악·통화 내용
- 사용자 단말의 다른 앱 사용 기록
- 광고 식별자 (AAID)

### 2.3 처리 방법

- 본 앱은 무선 이어폰이 송출하는 BLE 광고 신호(평문 공개 신호)를 수신해 화면에 표시할 뿐, 이어폰과 별도의 보안 채널을 형성하거나 이어폰의 저장 공간에 접근하지 않습니다.
- 위치 정보는 사용자가 "마지막 위치 보기" 기능을 명시적으로 활성화한 경우에만, 이어폰과의 연결이 끊어지는 시점에 1회 단발성으로 수집됩니다. 백그라운드에서 지속적으로 위치를 추적하지 않습니다.
- 모든 데이터는 사용자 단말 내 안드로이드 DataStore에 암호화되어 저장됩니다.

## 3. 처리의 법적 근거

| 정보 | 법적 근거 (개인정보 보호법 §15) |
|---|---|
| BLE 광고 패킷 | 정보주체의 동의 (앱 설치·실행이 곧 동의) |
| 마지막 위치 좌표 | 정보주체의 별도 동의 (위치 권한 동의 시점) |
| 앱 설정값 | 계약(서비스 제공)의 이행 |

---

## 4. 제3자 제공 및 위탁

**본 앱은 사용자의 개인정보를 제3자에게 제공하거나 위탁하지 않습니다.**

단, 향후 다음 서비스를 도입할 경우, 본 방침을 사전에 개정해 명시합니다.

- **Google Firebase Crashlytics** (도입 검토 중) — 앱 충돌 발생 시 익명화된 충돌 보고서를 Google LLC로 전송. 사용자 식별 정보·위치·페어링 이력은 포함되지 않으며, 앱 설정에서 옵트아웃 가능. (도입 시 개정 고지)

---

## 5. 보유·이용 기간

- 화면 표시용 BLE 광고 데이터. **즉시 폐기** (영속화하지 않음)
- 마지막 위치 좌표. **사용자가 삭제하거나 앱을 제거할 때까지**
- 앱 설정값. **앱을 제거할 때까지**

앱을 제거하면 단말 내 모든 데이터가 자동으로 삭제됩니다. 별도 서버에 저장되는 데이터가 없으므로 서버 측 삭제 절차는 존재하지 않습니다.

---

## 6. 사용자(정보주체)의 권리

사용자는 다음 권리를 행사할 수 있습니다.

1. **열람권.** 앱 내 "설정 → 저장된 데이터 보기"에서 마지막 위치 좌표를 직접 확인할 수 있습니다.
2. **정정·삭제권.** 앱 내 "설정 → 마지막 위치 삭제" 또는 앱 제거로 즉시 삭제할 수 있습니다.
3. **처리 정지권.** 앱 내 "설정 → 위치 기능 끄기" 또는 단말 설정에서 위치 권한을 회수해 처리를 중단시킬 수 있습니다.
4. **동의 철회권.** 앱을 제거하거나 단말 설정에서 권한을 회수하면 즉시 동의가 철회됩니다.

위 권리 행사로 인한 어떠한 불이익도 없습니다.

---

## 7. 권한 사용 목적

본 앱이 요청하는 안드로이드 권한과 그 목적은 다음과 같습니다.

| 권한 | 목적 | 거부 시 영향 |
|---|---|---|
| `BLUETOOTH_SCAN` (위치 정보 사용 안 함 플래그 부착) | 무선 이어폰의 BLE 광고 수신 | 본 앱의 핵심 기능 미동작 |
| `BLUETOOTH_CONNECT` | 이어폰 연결 상태 확인 | 연결 상태 표시 X |
| `FOREGROUND_SERVICE` + `FOREGROUND_SERVICE_CONNECTED_DEVICE` | 앱 종료 후에도 이어폰 상태 추적 | 앱이 백그라운드로 가면 기능 중단 |
| `POST_NOTIFICATIONS` (Android 13+) | 케이스 오픈 알림, 배터리 경고 | 알림 표시 X |
| `USE_FULL_SCREEN_INTENT` (Android 14+) | 케이스 오픈 풀스크린 알림 | 케이스 알림 표시 X |
| `ACCESS_FINE_LOCATION` (선택) | 마지막 위치 기록 (사용자 활성화 시) | "마지막 위치 보기" 기능 미동작 (다른 기능은 정상) |
| `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` | 삼성 절전 정책에서 백그라운드 동작 유지 | One UI 절전으로 인한 기능 중단 가능 |
| `RECEIVE_BOOT_COMPLETED` | 단말 재부팅 후 자동 재시작 | 재부팅 후 수동 실행 필요 |

특히 `BLUETOOTH_SCAN` 권한은 **`neverForLocation` 플래그**가 부착되어 있어, 안드로이드 시스템이 본 앱의 BLE 스캔으로부터 위치 정보를 추론하지 못하도록 차단되어 있습니다.

## 8. 14세 미만 아동의 개인정보

본 앱은 14세 미만 아동의 개인정보를 수집하지 않으며, 14세 미만 사용자는 본 앱의 위치 기능 동의 권한이 없습니다. 14세 미만 사용자가 위치 권한에 동의하려면 법정대리인의 동의가 필요합니다.

---

## 9. 개인정보의 안전성 확보 조치

- **암호화.** 단말 내 DataStore는 안드로이드 시스템이 제공하는 암호화 저장소를 사용합니다.
- **접근 제어.** 본 앱은 다른 앱이 본 앱의 데이터에 접근할 수 없도록 안드로이드 샌드박스 모델을 따릅니다.
- **외부 전송 차단.** 본 앱은 인터넷 통신 권한(`INTERNET`)을 사용하지 않습니다 (Crashlytics 도입 시 변경).
